Для тестирования использовались библиотеки: mocha (основной функционал), sinon (для тестирования console.log), chai (для функций asser, expect)
Весь функционал задания находится в добавленном файле test.js
Для того чтобы запустить можно даже не делать npm i, подключил все скрипты напрямую (до этого могут доебаться, но можешь сказать что тебе просто так удобнее было).
Следовательно, для запуска можно просто index.html в браузере открыть

По функционалу:
logStub, errorStub, warnStub нужны для того чтобы чекать вывод сообщений console.log, console.error, console.warn соответственно.
describe - это некий набор тестов для тестирования какой то одной задачи (в нашем случае было решено делать 1 блок describe для 1 файла. Пример: блок describe для cashbox, блок describe для catVasiliy и т.д.)

В основном для тестирования использовались следующие функции:
1) expect для проверки того что лежит в каком либо свойстве
Например:
expect(catVasiliy.name).to.eql("Василий");
Типа чекаем что свойство name у catVasiliy равно "Василий" (eql это "равно")
Замечание: могут спросить почему использовалось to.eql а не обычное to.equal. Скажи что to.equal херово работает при сравнении массивов
2) sinon.assert.calledWith - проверка того что в console вызывлся вывод сообщения (log, error, warn)
3) assert использовался для проверки функций, которые возвращают либо true либо false мы могли бы конечно снова замутить это через to.eql но нахера, если так более коротко и понятно)
